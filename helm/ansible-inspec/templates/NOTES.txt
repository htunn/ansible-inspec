1. Get the application URL by running these commands:
{{- if .Values.ingress.enabled }}
{{- range $host := .Values.ingress.hosts }}
  {{- range .paths }}
  http{{ if $.Values.ingress.tls }}s{{ end }}://{{ $host.host }}{{ .path }}
  {{- end }}
{{- end }}
{{- else if contains "NodePort" .Values.service.type }}
  export NODE_PORT=$(kubectl get --namespace {{ .Release.Namespace }} -o jsonpath="{.spec.ports[0].nodePort}" services {{ include "ansible-inspec.fullname" . }})
  export NODE_IP=$(kubectl get nodes --namespace {{ .Release.Namespace }} -o jsonpath="{.items[0].status.addresses[0].address}")
  echo http://$NODE_IP:$NODE_PORT
{{- else if contains "LoadBalancer" .Values.service.type }}
     NOTE: It may take a few minutes for the LoadBalancer IP to be available.
           You can watch the status of by running 'kubectl get --namespace {{ .Release.Namespace }} svc -w {{ include "ansible-inspec.fullname" . }}'
  export SERVICE_IP=$(kubectl get svc --namespace {{ .Release.Namespace }} {{ include "ansible-inspec.fullname" . }} --template "{{"{{ range (index .status.loadBalancer.ingress 0) }}{{.}}{{ end }}"}}")
  echo http://$SERVICE_IP:{{ .Values.service.port }}
{{- else if contains "ClusterIP" .Values.service.type }}
  export POD_NAME=$(kubectl get pods --namespace {{ .Release.Namespace }} -l "app.kubernetes.io/name={{ include "ansible-inspec.name" . }},app.kubernetes.io/instance={{ .Release.Name }}" -o jsonpath="{.items[0].metadata.name}")
  export CONTAINER_PORT=$(kubectl get pod --namespace {{ .Release.Namespace }} $POD_NAME -o jsonpath="{.spec.containers[0].ports[0].containerPort}")
  echo "Visit http://127.0.0.1:8080 to use your application"
  kubectl --namespace {{ .Release.Namespace }} port-forward $POD_NAME 8080:$CONTAINER_PORT
{{- end }}

2. Access the API documentation:
   http://YOUR_HOST/docs (Swagger UI)
   http://YOUR_HOST/redoc (ReDoc)

3. Check application health:
   http://YOUR_HOST/health

4. Get the admin credentials (if using default auth):
   Username: admin
   Password: Check secrets or set via values.yaml

{{- if .Values.postgresql.enabled }}

5. Database connection (PostgreSQL):
   Host: {{ include "ansible-inspec.fullname" . }}-postgresql
   Port: 5432
   Database: {{ .Values.postgresql.auth.database }}
   Username: {{ .Values.postgresql.auth.username }}
   
   To connect to the database from outside:
   kubectl port-forward --namespace {{ .Release.Namespace }} svc/{{ include "ansible-inspec.fullname" . }}-postgresql 5432:5432 &
   PGPASSWORD="{{ .Values.postgresql.auth.password }}" psql --host 127.0.0.1 -U {{ .Values.postgresql.auth.username }} -d {{ .Values.postgresql.auth.database }} -p 5432
{{- end }}

6. View application logs:
   kubectl logs -f --namespace {{ .Release.Namespace }} -l app.kubernetes.io/name={{ include "ansible-inspec.name" . }}

7. For authentication setup, refer to:
   - Azure AD: Set AUTH__AZURE_* environment variables
   - Local auth: Default enabled, use /auth/register to create users
   - RBAC: Supports admin, operator, and viewer roles

8. Important security notes:
   - Change default secrets in production!
   - Enable TLS on ingress
   - Review NetworkPolicy settings
   - Set up backup for persistent volumes

For more information, visit: https://github.com/htunn/ansible-inspec
